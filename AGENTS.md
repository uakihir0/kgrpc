# Documentation for Agents

## Overview

This repository is a Kotlin Multiplatform gRPC client library. It provides a common API across JVM, JavaScript, iOS, macOS, Linux, and Windows platforms.

## Directory Structure

- **`core/`**: Core gRPC client library (Channel, Stub, Message, Interceptors, RPC implementations)
- **`all/`**: Apple aggregation module (XCFramework, CocoaPods, SwiftPM)
- **`native/`**: Rust/Tonic native gRPC library compiled as static library for Kotlin/Native cinterop
- **`plugins/`**: Gradle convention plugins for publishing
- **`tool/`**: Build helper scripts (CocoaPods post-processing)

## Platform Implementations

| Platform | Source Set | gRPC Implementation | Streaming Support |
|----------|-----------|-------------------|------------------|
| JVM | jvmMain | grpc-java + grpc-kotlin-stub + grpc-okhttp | All types |
| JS (IR) | jsMain | gRPC-Web over Ktor HttpClient | Unary + Server-streaming |
| iOS / macOS | nativeMain | Rust/Tonic FFI via cinterop | All types |
| Linux x64 | nativeMain | Rust/Tonic FFI via cinterop | All types |
| Windows mingwX64 | nativeMain | Rust/Tonic FFI via cinterop | All types |

## Key Concepts

### Channel

`Channel` is the main entry point for creating gRPC connections. Use `Channel.Builder` to configure:

```kotlin
val channel = Channel.Builder
    .forAddress("localhost", 50051)
    .usePlaintext()
    .build()
```

### Message and MessageCompanion

All gRPC messages implement `Message` (serialize) and have a companion implementing `MessageCompanion<T>` (deserialize). These are typically generated by the protobuf code generation plugin.

### Stub

Generated service stubs extend `Stub<S>` and use the RPC functions (`unaryRpc`, `serverStreamingRpc`, etc.) to make calls.

### CallInterceptor

Interceptors can modify metadata, messages, and status at various points in the RPC lifecycle.

## Testing Method

Run JVM tests (no external dependencies required):

```shell
./gradlew core:jvmTest
```

Build verification (no network access needed):

```shell
./gradlew jvmJar
```

## Building Native Libraries

Rust toolchain is required for native platform builds:

```shell
# Build for current macOS platform
cd native && cargo build --release

# Build for all Apple targets
cd native && bash compile.sh

# Cross-compile for Linux/Windows (requires `cross` tool)
cd native && cross build --target x86_64-unknown-linux-gnu --release
cd native && cross build --target x86_64-pc-windows-gnu --release
```

## Implementation Guidelines

### Package Structure

All code lives under `work.socialhub.kgrpc`:
- `work.socialhub.kgrpc` - Core types (Channel, Code, Status, etc.)
- `work.socialhub.kgrpc.config` - Configuration types (KeepAliveConfig)
- `work.socialhub.kgrpc.message` - Message interfaces
- `work.socialhub.kgrpc.metadata` - Metadata types (Key, Entry, Metadata)
- `work.socialhub.kgrpc.rpc` - RPC implementation (per-platform)
- `work.socialhub.kgrpc.stub` - Stub base class
- `work.socialhub.kgrpc.internal` - JVM internal helpers
- `work.socialhub.kgrpc.util` - Utilities (PEM parsing)

### expect/actual Pattern

`Channel` uses expect/actual for platform-specific implementations:
- `commonMain/Channel.kt` - expect declaration
- `jvmMain/Channel.jvm.kt` - grpc-java ManagedChannel wrapper
- `jsMain/Channel.js.kt` - Ktor HttpClient with gRPC-Web
- `nativeMain/Channel.native.kt` - Rust/Tonic FFI

### Testing

- Place common tests in `core/src/commonTest/kotlin/`
- Place JVM-specific tests in `core/src/jvmTest/kotlin/`
- Test classes use kotlin.test annotations
